# 模块审查提示词：后端图片生成管道

## 模块概述

图片生成管道是系统的核心 AI 能力层，负责品类匹配、模型选择、LLM 智能分析、Prompt 构建和多风格图片生成任务编排。支持优雅降级（LLM 失败时回退到模板）。

## 涉及文件

| 文件 | 职责 | 行数 |
|------|------|------|
| `backend/prompts_v2.py` | Prompt 模板引擎、品类匹配、模型配置、国家配置 | ~892 |
| `backend/llm_provider.py` | LLM API 调用（阿里云百炼/Qwen VL） | 124 |
| `backend/llm_prompts.py` | LLM 系统/用户提示词模板 | ~150 |
| `backend/llm_schema.py` | LLM 输出 Pydantic Schema | 26 |

## 审查提示词

请对图片生成管道进行全面代码审查，重点检查以下方面：

### 1. Prompt 安全（Critical）

- **注入防护**：
  - Prompt 注入关键词黑名单是否充分（常见攻击向法）
  - 用户输入（商品类型、国家）是否可能绕过消毒逻辑
  - LLM 输出是否需要进一步验证（防止 LLM 被操纵）

- **输入验证**：
  - `match_category` 的输入是否需要白名单
  - `select_model` 的模型名称是否需要验证
  - 国家代码是否需要标准化

- **LLM 输出验证**：
  - LLM 返回的 JSON 是否经过 Pydantic 严格校验
  - 是否存在 LLM 返回恶意内容的风险
  - `LLMOutput` Schema 是否有遗漏的字段验证

### 2. LLM 集成（High）

- **降级策略**：
  - LLM 失败时回退到模板是否可靠
  - 降级后的 prompt 质量是否可接受
  - 是否应该区分临时故障和永久故障

- **API 调用**：
  - 超时设置（30s）是否合理（LLM 可能需要更长时间）
  - 是否需要重试机制
  - 错误分类（超时 vs 连接失败 vs 解析失败）

- **输出解析**：
  - `json.loads(content)` 是否处理了所有边界情况
  - `data["choices"][0]["message"]["content"]` 的空值处理
  - 嵌套 JSON 解析的异常处理

### 3. Prompt 质量（Medium）

- **品类匹配**：
  - 20 个品类是否覆盖所有商品类型
  - 模糊匹配的准确率
  - 自定义品类的处理

- **国家配置**：
  - 10 个国家的本地化描述是否准确
  - 货币、语言、购物习惯的配置是否完整
  - 是否需要支持更多国家

- **模型风格**：
  - 4 种 AI 模型的描述是否准确
  - 风格关键词是否有效
  - 模型选择的默认策略

### 4. 任务编排（Medium）

- **任务生成**：
  - `generate_all_tasks()` 生成 14 个任务（11 风格 + 白底 + 对比 + 细节）的逻辑是否正确
  - 任务间的依赖关系
  - 并发控制策略

- **Prompt 构建**：
  - 多维度 prompt 的组合逻辑
  - LLM 覆盖 prompt 的优先级
  - 风格变体的多样性

### 5. 代码质量（Medium）

- **文件大小**：
  - `prompts_v2.py` 约 892 行，是否需要拆分
  - 建议拆分：品类配置、模型配置、国家配置、prompt 构建器

- **硬编码配置**：
  - 品类、模型、国家配置是否应该外部化（JSON/YAML）
  - 配置更新是否需要重启服务

- **类型安全**：
  - LLM 输出的类型标注
  - 配置数据的类型验证

### 6. 可观测性（Low）

- **日志**：
  - LLM 调用的输入/输出是否记录
  - 降级事件是否记录
  - Prompt 构建的调试信息

- **指标**：
  - LLM 调用成功率
  - 降级率
  - 平均响应时间

## 审查输出格式

请按以下格式输出审查结果：

```
### [严重程度] 问题标题

**文件**: `backend/xxx.py:行号`
**问题**: 问题描述
**建议**: 改进方案
**影响**: 不修改的潜在风险
```

严重程度分级：
- **Critical**: 安全漏洞或数据损坏风险
- **High**: 功能缺陷或可靠性问题
- **Medium**: 代码质量或可维护性问题
- **Low**: 最佳实践建议
