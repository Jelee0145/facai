# 模块审查提示词：前端管理后台

## 模块概述

管理后台（`/admin`）提供系统管理功能，包括仪表盘、API Key 管理、生成历史、账单管理和 LLM 配置。独立布局，使用 JWT 认证和 CSRF 保护。

## 涉及文件

| 文件 | 职责 | 行数 |
|------|------|------|
| `src/app/admin/layout.tsx` | 管理后台布局（侧边栏、认证守卫） | ~150 |
| `src/app/admin/page.tsx` | 仪表盘（Key 健康、生成统计） | ~200 |
| `src/app/admin/error.tsx` | 管理后台错误边界 | ~30 |
| `src/app/admin/auth-context.tsx` | 管理认证上下文（JWT、fetchWithAuth） | ~200 |
| `src/app/admin/login/page.tsx` | 管理登录表单 | ~80 |
| `src/app/admin/keys/page.tsx` | API Key 管理（CRUD、启用/禁用） | ~300 |
| `src/app/admin/history/page.tsx` | 生成历史（分页、详情弹窗） | ~300 |
| `src/app/admin/billing/page.tsx` | 账单管理（套餐、订单、成本配置） | ~300 |
| `src/app/admin/llm/page.tsx` | LLM 配置（API Key、模型选择、测试） | ~200 |

## 审查提示词

请对前端管理后台进行全面代码审查，重点检查以下方面：

### 1. 权限控制与安全（Critical）

- **JWT 管理**（`auth-context.tsx`）：
  - Token 存储位置（cookie vs localStorage）是否安全
  - Token 过期后的自动刷新逻辑
  - `fetchWithAuth` 是否正确处理 401 响应（跳转登录）
  - 多个并发请求同时收到 401 时的竞态处理

- **路由守卫**（`layout.tsx`）：
  - 未认证用户是否能访问 `/admin` 子页面
  - 客户端路由守卫是否可被绕过（直接访问 URL）
  - 是否应该在 middleware 层做认证检查

- **CSRF 保护**：
  - CSRF token 是否在每次请求中正确携带
  - Token 是否与后端验证逻辑一致

### 2. 敏感数据处理（High）

- **API Key 展示**：
  - API Key 是否在前端被完整显示（应该掩码显示）
  - 创建/更新 API Key 时的明文传输
  - 浏览器历史记录中是否可能缓存敏感信息

- **LLM 配置**：
  - DashScope API Key 是否在前端明文显示
  - 测试连接时的密钥传输

- **密码处理**：
  - 登录表单的密码传输是否通过 HTTPS
  - 是否有密码强度要求

### 3. 数据管理（Medium）

- **分页与筛选**：
  - 历史记录分页的 URL 状态管理（是否支持分享链接）
  - 筛选条件的重置逻辑
  - 大数据量下的分页性能

- **表单验证**：
  - API Key CRUD 的输入验证
  - 账单配置的数值验证
  - 表单提交的防重复点击

- **数据刷新**：
  - 各页面的数据获取时机
  - 缓存策略
  - 手动刷新机制

### 4. UX 一致性（Medium）

- **交互模式**：
  - 删除操作的确认弹窗是否统一
  - 成功/错误提示的展示方式
  - 表格排序和搜索的交互

- **状态管理**：
  - Loading 状态的展示
  - 空数据状态的展示
  - 错误状态的恢复

- **响应式设计**：
  - 侧边栏在小屏幕下的表现
  - 表格的横向滚动

### 5. 代码质量（Low）

- **重复代码**：
  - 各页面的数据获取逻辑是否可以抽取为共享 Hook
  - 表格组件是否可以复用

- **类型安全**：
  - API 响应是否定义了 TypeScript 类型
  - 是否存在 `any` 类型

## 审查输出格式

请按以下格式输出审查结果：

```
### [严重程度] 问题标题

**文件**: `path/to/file.tsx:行号`
**问题**: 问题描述
**建议**: 改进方案
**影响**: 不修改的潜在风险
```

严重程度分级：
- **Critical**: 安全漏洞或权限绕过
- **High**: 数据泄露或功能缺陷
- **Medium**: 代码质量或 UX 问题
- **Low**: 最佳实践建议
