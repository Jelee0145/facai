> 本文档面向后端开发者，介绍项目结构、模块职责、配置说明和开发流程。

## 3. 环境变量

| 变量 | 必填 | 默认值 | 说明 |
|------|:----:|--------|------|
| `API_AUTH_TOKEN` | 生产必填 | `""` | 前后端内部认证令牌 |
| `JWT_SECRET` | 是 | — | JWT 签名密钥（≥32 字符随机值） |
| `CORS_ORIGINS` | 否 | `http://localhost:4524` | 允许的前端域名（逗号分隔） |
| `NODE_ENV` | 否 | `development` | 环境标识 |
| `COOKIE_SECURE` | 否 | `false` | Cookie 是否启用 Secure 标志 |
| `ADMIN_PASSWORD` | 首次启动 | — | 管理员初始密码 |
| `APIMART_INITIAL_WAIT_SECONDS` | 否 | `15` | 提交任务后首次查询前的等待时间（秒） |
| `APIMART_POLL_INTERVAL_SECONDS` | 否 | `4` | Apimart 任务轮询间隔（秒） |
| `APIMART_POLL_TIMEOUT_SECONDS` | 否 | `3600` | Apimart 任务总轮询超时（秒） |
| `SSE_IDLE_TIMEOUT_SECONDS` | 否 | `300` | SSE 空闲保活间隔（秒，空闲时发送 keep-alive） |

## 4. 开发环境搭建

### 首次安装

```bash
cd backend
pip install -r requirements.txt
```

### 配置 `.env`

```bash
cp .env.example .env
# 编辑 .env，填入 JWT_SECRET 等
```

### 启动开发服务器

```bash
cd backend
uvicorn main:app --host 0.0.0.0 --port 8001 --reload
```

- `--reload` 模式下代码修改自动重启
- 数据库文件 `data.db` 在首次启动时自动创建和初始化
- 管理员账号通过 `ADMIN_PASSWORD` 环境变量在启动时自动创建

### 运行测试

```bash
cd backend
python test_prompts.py          # Prompt 生成测试
python test_llm_provider.py     # LLM 调用测试
```

## 5. 数据库操作

### 连接管理

```python
from database import get_db

db = get_db()  # 返回线程本地 SQLite 连接
```

- 每个线程独立连接，通过 `threading.local()` 管理
- 自动启用 WAL 模式和外键约束
- `row_factory = sqlite3.Row`，支持 `row["column"]` 访问

### 新增表

在 `init_db()` 中添加 `CREATE TABLE IF NOT EXISTS` 语句。`init_db()` 在应用启动时由 `main.py` 的 `startup` 事件调用。

### 新增列（迁移）

```python
# 在 init_db() 末尾添加
existing_cols = [r[1] for r in db.execute("PRAGMA table_info('table_name')").fetchall()]
if "new_column" not in existing_cols:
    db.execute("ALTER TABLE table_name ADD COLUMN new_column TEXT DEFAULT ''")
```

### CRUD 函数

所有数据库操作封装在 `database.py` 中的函数里，`main.py` 通过 import 调用，不直接写 SQL。

## 6. 新增 API 端点指南

### 步骤 1：定义请求/响应模型

```python
# 在 main.py 中添加 Pydantic 模型
class MyRequest(BaseModel):
    name: str = Field(min_length=1, max_length=100)
    count: int = Field(default=1, ge=1, le=100)
```

### 步骤 2：添加数据库操作

```python
# 在 database.py 中添加 CRUD 函数
def get_my_items() -> list[dict]:
    db = get_db()
    rows = db.execute("SELECT * FROM my_table ORDER BY created_at DESC").fetchall()
    return [dict(r) for r in rows]
```

### 步骤 3：定义端点

```python
# 在 main.py 中添加端点
@app.get("/api/my-items")
async def list_my_items(user: dict = Depends(authenticate_customer)):
    return {"items": get_my_items()}

@app.post("/api/my-items")
@limiter.limit("20/minute")
async def create_my_item(
    request: Request,
    req: MyRequest,
    user: dict = Depends(authenticate_customer),
):
    _verify_user_csrf(request, user)
    item_id = add_my_item(sanitize_input(req.name, 100), req.count)
    return {"id": item_id}
```

### 认证选择

| 端点类型 | 认证方式 | CSRF |
|---------|---------|------|
| 公开端点 | 无 | 无 |
| 用户端点 | `Depends(authenticate_customer)` | `_verify_user_csrf()` |
| 管理端点 | `Depends(authenticate)` | `Depends(verify_csrf)` |
| 生成端点 | `Depends(verify_api_auth)` + `Depends(authenticate_customer)` | `_verify_user_csrf()` |

## 7. LLM 配置

LLM 功能通过 `system_config` 表运行时配置，无需重启服务。

| Config Key | 说明 | 默认值 |
|------------|------|--------|
| `llm_api_key` | DashScope API Key | 空（未配置时 LLM 分析跳过） |
| `llm_model` | 模型名称 | `qwen3-vl-flash` |

通过管理后台 `/admin/llm-config` 接口配置，支持测试连接。

### LLM 降级策略

LLM 分析失败时静默降级为模板 prompt，不影响图片生成主流程。降级日志：

```
[WARN] LLM analysis failed (downgrading to template): <error>
```

## 8. API Key 管理

### Key 生命周期

```
添加 Key → is_active=1, fail_count=0
  ↓
使用中 → 成功: today_used++, fail_count=max(fail_count-1, 0)
       → 失败(401/403): fail_count++
  ↓
fail_count ≥ 3 → is_active=0 (自动禁用)
  ↓
管理员重新启用 → is_active=1, fail_count=0
```

### 配额重置

`today_used` 需要外部定时任务调用 `reset_daily_usage()` 重置（每日零点）。

## 9. 启动流程

```
uvicorn main:app
  ↓
@app.on_event("startup")
  ↓
init_db()                    ← 建表 + 迁移
  ↓
检查 API_AUTH_TOKEN          ← 生产环境必须配置
  ↓
管理员账号检查/创建           ← ADMIN_PASSWORD 环境变量
  ↓
delete_old_tasks(24h)         ← 清理过期任务
  ↓
load_pending_tasks()          ← 恢复未完成任务（标记失败 + 退款）
  ↓
启动 reap_stale_tasks()       ← 每 60 秒回收僵死任务（>5 分钟无进展）
```

## 10. 错误处理

### 全局异常处理器

```python
@app.exception_handler(Exception)
async def global_exception_handler(request, exc):
    # 过滤敏感路径信息
    msg = msg.replace(os.getcwd(), "[PROJECT_ROOT]")
    # PromptValidationError → 400
    # HTTPException → 原样抛出
    # 其他 → 500 Internal server error
```

### 退款机制

生成任务失败时自动退回已扣积分：
- 同步生成：catch 块中调用 `_try_refund_generation()`
- 异步生成：后台协程 catch 块 + 僵死任务回收器
- 幂等保护：通过 `user_ledger` 查询防止重复退款
