# 数据库设计

> 本文档描述项目使用的 SQLite 数据库的完整 Schema、表间关系、索引策略和数据生命周期管理。

## 1. 数据库配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 引擎 | SQLite | 单文件数据库，文件路径 `backend/data.db` |
| 日志模式 | WAL (Write-Ahead Logging) | 支持并发读写，提升多线程性能 |
| 外键约束 | `PRAGMA foreign_keys=ON` | 启用外键引用完整性检查 |
| 连接模型 | 线程本地连接 (`threading.local`) | 每个线程独立连接，避免跨线程竞争 |
| 行工厂 | `sqlite3.Row` | 支持字典方式访问列名 |

### 迁移策略

项目不使用 ORM，采用**应用内增量迁移**：在 `init_db()` 中通过 `PRAGMA table_info()` 检测列是否存在，动态 `ALTER TABLE ADD COLUMN`。新增表使用 `CREATE TABLE IF NOT EXISTS`。

## 3. 表结构定义

### 3.1 api_keys — API Key 管理

存储 Apimart 图片生成 API 的密钥，支持多 Key 轮询和自动故障禁用。

```sql
CREATE TABLE api_keys (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    key_value     TEXT    NOT NULL UNIQUE,       -- 完整 API Key
    name          TEXT    DEFAULT '',             -- 可读名称
    is_active     INTEGER DEFAULT 1,             -- 1=启用, 0=禁用
    daily_limit   INTEGER DEFAULT 100,           -- 每日调用上限
    today_used    INTEGER DEFAULT 0,             -- 今日已用次数
    last_used_at  TEXT,                          -- 最近使用时间
    total_used    INTEGER DEFAULT 0,             -- 累计使用次数
    fail_count    INTEGER DEFAULT 0,             -- 连续失败次数（≥3 自动禁用）
    balance_usd   REAL    DEFAULT 0,             -- Key 余额（美元）
    total_balance_usd REAL DEFAULT 0,            -- 初始设置总额（美元），用于计算消耗百分比
    created_at    TEXT    DEFAULT (datetime('now')),
    updated_at    TEXT    DEFAULT (datetime('now'))
);
```

**业务规则：**
- 连续鉴权失败 ≥3 次自动设置 `is_active=0`
- 每日配额耗尽（`today_used ≥ daily_limit`）时跳过该 Key
- 重新启用（`is_active=1`）时自动重置 `fail_count=0`
- 手动更新余额（`balance_usd`）时自动重置 `today_used=0`、`total_used=0`

### 3.2 generation_history — 生成历史

记录每次图片生成任务的完整信息，包括 LLM 请求/响应详情。

```sql
CREATE TABLE generation_history (
    id                  INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id             TEXT    UNIQUE,               -- 任务 UUID
    api_key_id          INTEGER,                      -- 使用的 API Key (FK → api_keys)
    user_id             INTEGER,                      -- 前台用户 (FK → users, 迁移添加)
    product_type        TEXT    DEFAULT '',            -- 产品品类
    country             TEXT    DEFAULT '',            -- 目标国家
    model               TEXT    DEFAULT '',            -- 生成模型
    prompt_size         TEXT    DEFAULT '',            -- prompt 尺寸
    prompt_resolution   TEXT    DEFAULT '',            -- 生成分辨率
    total_images        INTEGER DEFAULT 14,           -- 总图片数
    success_count       INTEGER DEFAULT 0,            -- 成功图片数
    status              TEXT    DEFAULT 'pending',     -- pending/generating/completed/error
    elapsed_seconds     REAL    DEFAULT 0,            -- 耗时（秒）
    error_msg           TEXT    DEFAULT '',            -- 错误信息
    llm_request         TEXT    DEFAULT '',            -- LLM 请求详情 (JSON)
    llm_response        TEXT    DEFAULT '',            -- LLM 响应详情 (JSON)
    tasks_detail        TEXT    DEFAULT '',            -- 子任务详情 (JSON)
    charge_points       INTEGER DEFAULT 0,            -- 扣除积分
    description_snapshot TEXT   DEFAULT '',            -- 产品描述快照
    preview_images_json TEXT    DEFAULT '',            -- 预览图 URL (JSON array)
    created_at          TEXT    DEFAULT (datetime('now')),
    FOREIGN KEY (api_key_id) REFERENCES api_keys(id),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 3.3 admin_users — 管理后台用户

```sql
CREATE TABLE admin_users (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    username        TEXT    UNIQUE NOT NULL,
    password_hash   TEXT    NOT NULL,              -- bcrypt 哈希
    role            TEXT    DEFAULT 'admin',
    is_active       INTEGER DEFAULT 1,
    last_login      TEXT,
    login_attempts  INTEGER DEFAULT 0,             -- 连续失败次数
    locked_until    TEXT,                          -- 锁定截止时间
    created_at      TEXT    DEFAULT (datetime('now'))
);
```

**业务规则：**
- 连续登录失败 ≥5 次锁定 30 分钟

### 3.6 users — 前台注册用户

```sql
CREATE TABLE users (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    username        TEXT    UNIQUE NOT NULL,
    password_hash   TEXT    NOT NULL,              -- bcrypt 哈希
    phone           TEXT    DEFAULT '',
    email           TEXT    DEFAULT '',
    status          TEXT    DEFAULT 'active',      -- active/disabled
    is_unlimited    INTEGER DEFAULT 0,             -- 1=无限额度（不扣积分）
    login_attempts  INTEGER DEFAULT 0,
    locked_until    TEXT,
    last_login      TEXT,
    created_at      TEXT    DEFAULT (datetime('now')),
    updated_at      TEXT    DEFAULT (datetime('now'))
);
```

**业务规则：**
- `is_unlimited=1` 的用户生成不扣积分，消费记录记为 0
- 连续登录失败 ≥5 次锁定 30 分钟

### 3.9 orders — 充值订单

```sql
CREATE TABLE orders (
    id                INTEGER PRIMARY KEY AUTOINCREMENT,
    order_no          TEXT    UNIQUE NOT NULL,     -- 订单号 (PAY 前缀)
    user_id           INTEGER NOT NULL,           -- 用户 (FK → users)
    package_id        INTEGER,                    -- 套餐 (FK → credit_packages)
    amount_fen        INTEGER NOT NULL,           -- 金额（分）
    points            INTEGER NOT NULL,           -- 实际获得积分（基础+赠送）
    status            TEXT    DEFAULT 'pending',   -- pending/submitted/rejected/paid/credited
    payment_provider  TEXT    DEFAULT 'manual',    -- 支付渠道 (manual=个人收款码)
    created_at        TEXT    DEFAULT (datetime('now')),
    paid_at           TEXT,                       -- 支付时间
    credited_at       TEXT,                       -- 到账时间
    payment_remark    TEXT    DEFAULT '',          -- 用户付款备注（微信昵称/转账附言）
    proof_image       TEXT    DEFAULT '',          -- 付款截图路径 (/uploads/proofs/xxx.png)
    submitted_at      TEXT,                       -- 用户提交凭证时间
    reviewed_at       TEXT,                       -- 管理员审核时间
    reviewer_note     TEXT    DEFAULT '',          -- 管理员审核备注
    reject_reason     TEXT    DEFAULT '',          -- 驳回原因
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (package_id) REFERENCES credit_packages(id)
);
```

**状态流转：** `pending` → `submitted` → `credited`（正常）
驳回流程：`submitted` → `rejected` → 用户重新提交 → `submitted`

### 3.11 user_ledger — 用户积分流水

记录所有积分变动，是余额准确性的唯一来源。

```sql
CREATE TABLE user_ledger (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id         INTEGER NOT NULL,              -- 用户 (FK → users)
    type            TEXT    NOT NULL,              -- recharge/consume/refund
    direction       TEXT    NOT NULL,              -- in（入账）/out（扣减）
    points          INTEGER NOT NULL,              -- 变动积分
    balance_after   INTEGER NOT NULL,              -- 变动后余额
    order_id        INTEGER,                      -- 关联订单 (FK → orders, 充值时)
    reference_id    TEXT    DEFAULT '',            -- 关联 ID（task_id 等）
    remark          TEXT    DEFAULT '',            -- 备注
    created_at      TEXT    DEFAULT (datetime('now')),
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (order_id) REFERENCES orders(id)
);
```

**类型说明：**

| type | direction | 场景 |
|------|-----------|------|
| `recharge` | `in` | 充值到账 |
| `consume` | `out` | 生成扣费 |
| `refund` | `in` | 生成失败退款 |

**幂等性：** 退款操作通过查询 ledger 中是否已有相同 `reference_id` 的 `refund` 记录防止重复退款。

### 3.12 task_store — 异步任务持久化

存储异步图片生成任务的状态和数据，用于服务重启后恢复未完成任务。

```sql
CREATE TABLE task_store (
    task_id     TEXT PRIMARY KEY,                  -- 任务 UUID
    data        TEXT    NOT NULL,                  -- 任务完整数据 (JSON)
    status      TEXT    DEFAULT 'submitting',      -- submitting/generating/completed/error
    created_at  TEXT    DEFAULT (datetime('now')),
    updated_at  TEXT    DEFAULT (datetime('now'))
);
```

**清理策略：** 超过 24 小时的任务记录自动删除（`delete_old_tasks()`）。

---

### 3.13 customer_login_attempts — 前台用户登录尝试

针对**不存在的用户名**也能记录登录尝试，防止用户名枚举攻击。

```sql
CREATE TABLE customer_login_attempts (
    username        TEXT PRIMARY KEY,
    login_attempts  INTEGER DEFAULT 0,
    locked_until    TEXT,
    updated_at      TEXT    DEFAULT (datetime('now'))
);
```

---

### 3.14 revoked_tokens — 已吊销 JWT

存储被主动吊销的 JWT Token 的 `jti`（JWT ID）声明。

```sql
CREATE TABLE revoked_tokens (
    jti         TEXT PRIMARY KEY,
    expires_at  TEXT    NOT NULL,                  -- Token 原始过期时间
    revoked_at  TEXT    DEFAULT (datetime('now'))
);
```

**清理策略：** 过期的吊销记录由 `cleanup_expired_revoked_tokens()` 清理。

## 5. 索引

```sql
-- 生成历史高频查询
CREATE INDEX idx_history_user   ON generation_history(user_id);
CREATE INDEX idx_history_created ON generation_history(created_at);

-- 积分流水高频查询
CREATE INDEX idx_ledger_user    ON user_ledger(user_id);
CREATE INDEX idx_ledger_created ON user_ledger(created_at);

-- 异步任务状态查询
CREATE INDEX idx_task_status    ON task_store(status);
CREATE INDEX idx_task_created   ON task_store(created_at);

-- Token 吊销清理
CREATE INDEX idx_revoked_expires ON revoked_tokens(expires_at);
```

## 6. 数据生命周期

| 数据 | 保留策略 | 清理方式 |
|------|---------|---------|
| `task_store` | 24 小时 | `delete_old_tasks(hours=24)` |
| `revoked_tokens` | 过期后可清理 | `cleanup_expired_revoked_tokens()` |
| `api_keys.today_used` | 每日重置 | `reset_daily_usage()` |
| `generation_history` | 永久保留 | — |
| `user_ledger` | 永久保留 | — |
| `orders` | 永久保留 | — |

## 7. 初始数据

应用首次启动时自动插入：

1. **充值套餐**（3 条）：体验包、标准包、增长包
2. **系统配置**：`generation_cost_points = "10"`

管理员账号需通过 seed 脚本或 API 创建，不在 `init_db()` 中自动创建。
