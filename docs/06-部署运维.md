## 2. Docker Compose 配置

### 服务概览

| 服务 | 镜像 | 端口 | 内存限制 | 健康检查 |
|------|------|------|---------|---------|
| `backend` | python:3.11-slim | 8001 | 512MB | `GET /` 每 30s |
| `frontend` | node:22-alpine (multi-stage) | 4524 | 512MB | — |

### 依赖关系

```yaml
frontend:
  depends_on:
    backend:
      condition: service_healthy  # 后端健康后才启动前端
```

### 持久化

```yaml
volumes:
  - ./backend/data.db:/app/data.db  # SQLite 数据库挂载
```

### 环境变量传递

| 服务 | 来源 | 变量 |
|------|------|------|
| backend | `./backend/.env` | 全部后端变量 |
| backend | docker-compose environment | `CORS_ORIGINS`, `COOKIE_SECURE` |
| frontend | docker-compose environment | `BACKEND_URL=http://backend:8001` |

## 3. Dockerfile 说明

### Backend (`Dockerfile.backend`)

```dockerfile
FROM python:3.11-slim
WORKDIR /app
RUN adduser --system --group appuser   # 非 root 用户
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
RUN chown -R appuser:appuser /app
USER appuser                             # 以非 root 运行
EXPOSE 8001
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8001"]
```

### Frontend (`Dockerfile.frontend`)

**多阶段构建：**

| 阶段 | 基础镜像 | 操作 |
|------|---------|------|
| builder | node:22-alpine | pnpm install → next build → tsup 打包 server.ts |
| runner | node:22-alpine | 仅复制产物 + 生产依赖 |

**构建产物：**
- `.next/` — Next.js 编译输出
- `dist/server.js` — 自定义 HTTP Server（tsup 打包）
- `public/` — 静态资源

**端口：** 容器内监听 4524（`PORT=4524`），docker-compose 映射 `4524:4524`。

## 4. 环境变量清单

### 后端 (`backend/.env`)

| 变量 | 必填 | 默认值 | 说明 |
|------|:----:|--------|------|
| `APIMART_API_KEY` | 是 | — | Apimart API Key |
| `JWT_SECRET` | 是 | — | JWT 签名密钥（≥32 字符） |
| `API_AUTH_TOKEN` | 生产必填 | `""` | 前后端内部认证令牌 |
| `ADMIN_PASSWORD` | 首次 | — | 管理员初始密码 |
| `CORS_ORIGINS` | 否 | `http://localhost:4524` | 允许的前端域名 |
| `NODE_ENV` | 否 | `development` | 环境标识 |
| `COOKIE_SECURE` | 否 | `false` | Cookie Secure 标志 |

### 前端

| 变量 | 必填 | 默认值 | 说明 |
|------|:----:|--------|------|
| `BACKEND_URL` | 是 | `http://localhost:8001` | 后端服务地址 |
| `API_AUTH_TOKEN` | 是 | — | 内部认证令牌（与后端一致） |
| `PORT` | 否 | `4524` | 前端监听端口 |
| `HOSTNAME` | 否 | `localhost` | 监听地址 |
| `NODE_ENV` | 否 | `development` | 环境标识 |

### 生产环境生成密钥

```bash
# JWT_SECRET
python -c "import secrets; print(secrets.token_hex(32))"

# API_AUTH_TOKEN
python -c "import secrets; print(secrets.token_hex(32))"

# ADMIN_PASSWORD (满足密码策略: 12+位, 大小写+数字+特殊字符)
python -c "import secrets; print(secrets.token_urlsafe(16))"
```

## 5. 启动方式

### 方式一：Docker Compose（推荐）

```bash
# 构建并启动
docker compose up -d --build

# 查看日志
docker compose logs -f

# 停止
docker compose down
```

### 方式二：本地开发

```bash
# 终端 1: 启动后端
cd backend
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8001 --reload

# 终端 2: 启动前端
pnpm install
bash ./scripts/dev.sh
```

### 方式三：一键脚本 (Windows)

```bash
# PowerShell
.\scripts\up.ps1

# 或双击
一键启动.bat
```

## 7. 数据库备份

### 备份脚本 (`scripts/backup-db.sh`)

```bash
bash ./scripts/backup-db.sh
```

### 手动备份

```bash
# 复制 SQLite 文件（需确保无写入进行中，WAL 模式下可安全复制）
cp backend/data.db backup/data_$(date +%Y%m%d_%H%M%S).db
```

### 恢复

```bash
# 停止后端服务
docker compose stop backend

# 替换数据库文件
cp backup/data_20260530.db backend/data.db

# 重启
docker compose start backend
```

### 备份策略建议

- 每日自动备份到异地存储
- 保留最近 7 天的备份
- 每月验证备份可恢复性

## 8. 运维操作

### 重置管理员密码

```bash
# 停止后端
docker compose stop backend

# 设置新的 ADMIN_PASSWORD 环境变量后重启
# 后端启动时会检测密码哈希是否有效，无效时用 ADMIN_PASSWORD 修复
ADMIN_PASSWORD="NewStr0ng!Pass" docker compose start backend
```

### 重置每日 API Key 配额

```bash
# 需要外部 cron 或手动调用
# 后端提供 reset_daily_usage() 函数，但未暴露为 API 端点
# 可通过 Python 直接调用:
cd backend && python -c "from database import reset_daily_usage; reset_daily_usage()"
```

### 清理过期任务

```bash
# 自动执行：后端启动时清理 24 小时前的任务
# 手动执行:
cd backend && python -c "from database import delete_old_tasks; delete_old_tasks(24)"
```

### 查看日志

```bash
# Docker
docker compose logs -f backend
docker compose logs -f frontend

# 本地
# 后端日志输出到 stdout，格式: 2026-05-30 10:00:00 [INFO] message
```

## 10. 反向代理配置示例

### Nginx

```nginx
server {
    listen 443 ssl;
    server_name app.example.com;

    # 前端
    location / {
        proxy_pass http://localhost:4524;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 后端 API（如果需要直接暴露）
    location /api/backend/ {
        proxy_pass http://localhost:8001/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # SSE 支持
    location ~ /stream$ {
        proxy_pass http://localhost:8001;
        proxy_set_header Connection '';
        proxy_http_version 1.1;
        chunked_transfer_encoding off;
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
    }
}
```

### SSE 注意事项

SSE 端点需要特殊配置：
- 关闭 proxy buffering
- 关闭 chunked transfer encoding
- 设置较长的 read timeout（建议 300 秒）
- 使用 HTTP/1.1
