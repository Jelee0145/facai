## 4. 自定义 HTTP Server

`src/server.ts` 使用 Node.js `http.createServer` + Next.js 手动集成，而非 Next.js 内置 server。

**原因：** 需要对服务生命周期有更细粒度的控制（优雅关闭、信号处理等）。

**功能：**
- 监听 `PORT`（默认 4524）和 `HOSTNAME`（默认 localhost）
- SIGTERM / SIGINT 信号处理：5 秒内优雅关闭，超时强制退出
- 未处理 Promise 拒绝捕获

## 5. API 代理层

### 核心函数：`proxyToBackend()`

所有 `/api/*` 路由统一使用 `src/lib/proxy.ts` 中的 `proxyToBackend()` 函数。

**基本用法：**

```typescript
// src/app/api/auth/[...path]/route.ts
import { proxyToBackend } from "@/lib/proxy";

export async function POST(request: NextRequest, { params }: { params: { path: string[] } }) {
  return proxyToBackend(request, "POST", params, {
    targetPrefix: "/auth",
    forwardCookies: true,     // 登录需要转发 Cookie
    forwardSetCookie: true,   // 需要回传 Set-Cookie
  });
}

export async function GET(request: NextRequest, { params }: { params: { path: string[] } }) {
  return proxyToBackend(request, "GET", params, {
    targetPrefix: "/auth",
    forwardCookies: true,
  });
}
```

### 代理配置项

| 选项 | 类型 | 说明 |
|------|------|------|
| `targetPrefix` | string | 后端路径前缀，替换原始路径 |
| `forwardCookies` | boolean | 转发请求 Cookie 到后端 |
| `forwardSetCookie` | boolean | 回传后端 Set-Cookie 到浏览器 |
| `handleStreaming` | boolean | 支持 SSE 流式转发 |
| `customPath` | string | 自定义后端完整路径 |
| `allowedMethods` | string[] | 允许的 HTTP 方法 |

### 代理流程

```
浏览器请求
  → proxyToBackend()
    → 构建后端 URL (BACKEND_URL + path)
    → 注入 X-API-Auth 请求头
    → 过滤 hop-by-hop 头
    → 判断是否 SSE (Accept: text/event-stream 或 URL 以 /stream 结尾)
      → SSE: 直接 fetch + 透传 stream
      → 普通: fetchWithRetry + withCircuitBreaker
    → 解析 JSON 响应 → NextResponse.json()
```

### 各路由代理配置

| 路由 | targetPrefix | forwardCookies | forwardSetCookie | handleStreaming |
|------|:------------:|:--------------:|:----------------:|:--------------:|
| `/api/auth/*` | `/auth` | ✓ | ✓ | — |
| `/api/user/*` | `/user` | ✓ | — | — |
| `/api/admin/*` | `/admin` | ✓ | ✓ | — |
| `/api/generate/*` | `/api/generate` | ✓ | — | ✓ |
| `/api/custom-types` | `/api/custom-types` | — | — | — |

## 6. 网络韧性

### fetchWithRetry (`src/lib/fetch.ts`)

指数退避重试，仅对 502/503/504 状态码重试。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `maxRetries` | 3 | 最大重试次数 |
| `baseDelay` | 1000ms | 基础延迟 |
| 退避策略 | 指数 | 1s → 2s → 4s |

### CircuitBreaker (`src/lib/circuit-breaker.ts`)

熔断器保护，防止后端故障时前端持续请求导致雪崩。

```
CLOSED (正常)
  → 连续失败 ≥5 次
OPEN (熔断，30 秒)
  → 30 秒后
HALF_OPEN (试探)
  → 成功 → CLOSED
  → 失败 → OPEN
```

熔断时返回 `503 Service temporarily unavailable` + `Retry-After: 30`。

## 7. SSE 实时通信

### useSSE Hook (`src/lib/use-sse.ts`)

封装 EventSource 的完整生命周期管理。

**使用方式：**

```typescript
import { useSSE } from "@/lib/use-sse";

function MyComponent() {
  const [taskId, setTaskId] = useState<string | null>(null);
  const [progress, setProgress] = useState(0);

  useSSE(taskId, {
    onOpen: () => console.log("SSE connected"),
    onProgress: (data) => {
      setProgress(data.completed / data.total);
    },
    onComplete: (data) => {
      console.log("Done!", data.result);
      setTaskId(null);
    },
    onError: (error) => {
      console.error(error);
      setTaskId(null);
    },
  });

  return <div>进度: {Math.round(progress * 100)}%</div>;
}
```

**行为：**
- `taskId` 非空时自动创建 EventSource
- 断开后自动指数退避重连，直到任务终态或确认会话过期
- 重连时检查 `/auth/me` 验证会话有效性
- 重连后先接收当前任务快照，避免错过 `completed` 结果
- `taskId` 变为 null 或组件卸载时自动清理

## 8. UI 规范

### 组件库

优先使用 `src/components/ui/` 下的 shadcn/ui 组件，通过 `@/components/ui/...` 别名导入。

### 主题

- CSS 变量定义在 `src/app/globals.css`
- 使用 oklch 色彩空间
- Tailwind CSS v4 语法：`@import 'tailwindcss'` + `@theme`

### Hydration 安全

禁止在 JSX 中直接使用：
- `typeof window`
- `Date.now()`
- `Math.random()`

必须用 `useEffect` + `useState` 模式：

```typescript
const [mounted, setMounted] = useState(false);
useEffect(() => setMounted(true), []);
if (!mounted) return <Skeleton />;
```

### 包管理

仅使用 pnpm。`package.json` 中有 preinstall 脚本禁止 npm/yarn。

## 9. 开发规范

### TypeScript

- Strict 模式
- 禁止隐式 `any`、`as any`
- catch 子句必须收窄类型

### ESLint

- 禁止 `<head>` 标签（使用 `metadata` 导出）
- `next.config` 中禁止硬编码绝对路径
- `react-hooks/set-state-in-effect` 已关闭

### next.config

路径必须用 `path.resolve(__dirname, ...)` 或 `process.cwd()` 动态拼接。

## 10. 管理后台认证

管理后台使用 `AuthProvider` Context：

```
/admin/layout.tsx → AuthProvider
  → 初始化: GET /api/admin/me
  → 成功: setUser(response)
  → 401: setUser(null), 跳转 /admin/login
```

所有管理页面通过 `useAuth()` 获取用户状态。请求通过 `fetchWithAuth()` 自动注入 CSRF Token。
