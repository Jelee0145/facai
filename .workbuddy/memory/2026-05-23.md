# 2026-05-23 工作记录

## 修复后台管理界面登录后被踢回登录页的问题（全量修复）

**根本原因**: 前后端数据结构不匹配
- 后端 `/admin/login` 返回扁平结构 `{ access_token, username, role, ... }`
- 前端 `auth-context.tsx` 期望 `data.user` 对象，导致 `data.user` 为 `undefined`
- `localStorage.setItem("admin_user", JSON.stringify(undefined))` 写入字符串 `"undefined"`
- 页面跳转后 AuthProvider 重新初始化，`u === "undefined"` 校验不通过，token 状态不被恢复
- layout 检测到 `token === null`，重定向回登录页

**全量修复清单**:

### 后端
1. `backend/security.py`: login() 返回结构改为嵌套 `user: { username, role }` 对象
2. `backend/main.py`: 端口从 8000 改为 8001，与前端 API_BASE 一致

### 前端
3. `src/app/admin/auth-context.tsx`:
   - 初始化 useEffect: token 存在即恢复，user 异常不再连带清除 token
   - login 函数: 添加 `data.user || { username, role }` 兼容旧后端
   - fetchWithAuth: 使用 tokenRef 避免闭包过期，401 自动登出，Content-Type 按需设置
   - logout: 不再做 window.location.href 跳转，由 layout 统一处理
4. `src/app/admin/layout.tsx`:
   - 侧边栏 `<a>` 改为 `<Link>` 客户端导航
   - 新增已登录用户访问登录页自动跳转仪表盘
   - handleLogout 统一处理退出+跳转
5. `src/app/admin/login/page.tsx`: router.push 改为 router.replace 防止回退
6. `src/app/admin/page.tsx`: fetchWithAuth 添加 HTTP 状态码校验
7. `src/app/admin/keys/page.tsx`: load 和 addKey 添加错误处理
8. `src/app/admin/history/page.tsx`: load 添加 HTTP 状态码校验和防御性数据解析
